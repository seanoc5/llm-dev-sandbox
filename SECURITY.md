# Security Policy

This is a sandbox / sysadmin sub-project — a personal toolkit for orchestrating LLM workers, not a production service or a hosted product. There are no users to protect beyond the maintainer running it locally.

## Reporting a vulnerability

Please **do not** open a public issue for security reports.

Preferred channel: open a private security advisory via GitHub:
<https://github.com/seanoc5/llm-swarm-runner/security/advisories/new>

If GitHub private advisories are unavailable, reach out directly to the maintainer (`seanoc5` on GitHub) and request a private channel before disclosing details.

## What to expect

There is no formal SLA. Reports will be triaged on a best-effort basis. Given the project's scope (single-maintainer, local-sandbox tooling), fixes will typically arrive as ordinary PRs once a report is confirmed.
